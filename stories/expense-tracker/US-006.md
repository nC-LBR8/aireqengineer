# US-006: [CHAR] Maintainability - Expense logic separation
Status: todo
Priority: medium
Requirement-Refs: REQ-001
Feature-Refs: FEAT-001, FEAT-002, FEAT-003
Tags: NFR-Maintainability

## Description
Organize expense tracking, filtering, summaries, and local storage behavior so the app can be tested and extended without large rewrites.

## Acceptance Criteria
- Expense CRUD logic, filter logic, summary calculation, and local persistence are separated from presentation code.
- Automated tests cover adding an expense, editing or deleting it, and recalculating a monthly summary after a change.
- Project documentation explains how to run the app locally and where expense, filter, summary, and storage logic is implemented.
